## Good Practices for System Prompt Writing

### 1. **Structure & Organization**
- Use clear sections with headers (e.g., `====SECTION====` or `##`)
- Organize logically: Role → Capabilities → Rules → Instructions
- Use bullet points and numbered lists for clarity

### 2. **Role Definition**
- Start with a clear identity statement ("You are...")
- Define expertise areas and personality traits
- Set the tone and communication style

### 3. **Be Explicit About Constraints**
- State what the AI should NOT do
- Define boundaries and limitations
- Use emphatic language for critical rules ("NEVER", "ALWAYS", "MUST")

### 4. **Provide Examples**
- Include input/output examples for complex behaviors
- Show format templates for structured responses
- Demonstrate edge cases

### 5. **Context & Environment**
- Provide relevant system information
- Define working directory, tools available, etc.
- Explain the context the AI operates in

### 6. **Output Formatting**
- Specify response format (JSON, markdown, plain text)
- Define any special syntax requirements
- Set length expectations where relevant

### 7. **Error Handling**
- Define how to handle ambiguous requests
- Specify fallback behaviors
- Include instructions for requesting clarification

### 8. **Testing & Iteration**
- Test with edge cases
- Refine based on failure modes
- Version control your prompts

### Key Principle
**Specificity beats brevity** - It's better to be explicit and detailed than concise and ambiguous. AI will follow clear instructions more reliably than implied expectations.
